部署说明

  • Bi 的部署方式支持纯文本和链接两种方式，同时提供重复生成 JS 代码功能，用来更新探针版本。

应用设置

  • 应用设置由应用重命名设置、用户体验阈值设置、页面慢加载阈值设置、URL(Page,Iframe,AJAX)采集设置、总览关键地理区域设置 5 部分组成。如图：

应用重命名

  • 勾选是否隐藏掉该应用，隐藏掉后此应用在应用列表不可见；

  • 名称修改仅仅是为了显示，不会更改原始名称.

用户体验阈值设置

  • 定义用户满意度的区间：满意、可容忍（缓慢） 、不满意（严重缓慢）

  • 默认值：满意（0-2s） 、可容忍（2-8s） 、不满意（>8s）

慢加载阈值设置

  • 设置「慢加载追踪」模块的采集阈值；

  • 默认大于 8s 的页面加载采集页面详情数据

URL(Pages,iFrames,AJAX) 采集设置

  • 根据 URL 在浏览器不同表现形式，将 URL 划分为三种类型：页面 （Page） 、iFrames、AJAX;

  • Bi 对用户体验的监控完全依托于页面，所有页面 （Page）类型是必须采集的，相对应的 Iframes 和 AJAX 则可通过配置设置是否采集

  • URL 合并及命名规则

    • 规则包括三部分：规则名称和优先级、URL 过滤 、 URL 命名
    • 规则名称和优先级： 用户可以自定义新建规则，自定义规则要给规则起一个有含义的名称，当和其他规则有交集时要对比优先级，优先级分为 1-10，数字越大优先级越高，同级别规则以创建时间较早为优先；

应用系统会默认生成一条默认规则，且不可删除，不可改规则名称和优先级，默认规则优先级始终为 0，且始终生效，如下图，选项均为不可修改；

  • URL 过滤：

    • URL 字符字符过滤，匹配方式包括：包含、开始于、结束于、等于、正则匹配；

  • URL 命名：

如下示例，将 URL 命名分段定义如下： http://www. host.com:8080/p/a/t/h?query=string#hash 协议：http:// 域名：www. host.com:8080 urlPath：/p/a/t/h 参数采集：query=string 锚点：hash 根据以上对 URL 的定义，可以定制 URL 的命名也就是展示名称:

  • URL 不采集规则

不采集匹配到了 URL 数据将直接被丢弃
