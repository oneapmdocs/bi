访问页面

该页面为访问页面，共有过滤条件、搜索栏、事务展示栏 3 部分组成。

过滤条件

  • 点击过滤条件展示出过滤选择栏，由页面类型、访问量、页面加载时间 3 方面来进行对访问页面过滤。

搜索栏

  • 可以在搜索栏输入关键字来对访问页面的 URL 进行搜素。

事务栏展示

  • 访问页面是用户打开网页的唯一地址，这个地址的核心是 URL，同时 URL 也是 Bi 产品的核心时间； 对 URL 采集及命名参见「应用设置」模块的URL采集设置；

  • 点击单条 URL，可以查看详情；

  • 分性能详情、后端事务、AJAX 调用、慢加载追踪 4 部分。

性能详情

  • 由用户体验满意度和页面加载时间模块、访问量和 AJAX 调用量和页面 JS 错误数模块、Apdex 得分、访问量和页面加载时间模块、性能区间 5 个模块组成。

  • 用户体验满意度和页面加载时间模块展示的是当前 URL 的用户体验满意度和该页面加载时间。

  • 访问量和 AJAX 调用量和页面 JS 错误数模块展示的是当前 URL 的访问量、AJAX 调用量和页面 JS 错误数。

  • Apdex 得分展示的是当前 URL 在响应的时间段内的满意度。

  • 访问量和页面加载时间模块展示的是当前时间段访问量和页面加载时间随着时间变化的情况。

  • 性能区间展示的是当前 URL 的 Web 应用程序、网络、网页加载和资源加载的情况；

后端事务

  • 如果在应用服务器端部署了 Ai 探针，则会记录后端处理该 URL 的后端事务名称、处理时间等信息；通过后端事务名称可点击跳转到 Ai 进行下钻分析。

AJAX 调用

  • 记录发生在该页面下的所有 AJAX 请求，展示具体 AJAX 名称、AJAX 平均响应时间、调用量、AJAX 每分钟调用量

  • 点击单条的页面名称可以跳转到 AJAX 功能里展示详细信息，AJAX 功能在后面会详细讲解；

慢加载追踪

  • 访问该页面时所发生的单次缓慢加载快照，支持过滤条件查询；

  • 点击单条 URL 展示出访问时间 URL、IP、省份、城市、浏览器、页面加载时间、Web 应用耗时的甘特图。

  • 精度展示了该页面加载过程，包括以下指标：

    • Unload ：浏览器卸载前一个页面耗时；
    • Redirect：浏览器重定向耗时；
    • Cache：读取缓存耗时；
    • DNS：DNS 寻址耗时；
    • TCP: TCP 链接耗时；
    • Request：浏览器发送请求耗时；
    • Response：浏览器收到完整响应耗时；
    • Dom：Dom 结构构建耗时；
    • Page：页面渲染耗时。
