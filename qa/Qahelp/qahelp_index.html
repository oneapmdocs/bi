常见问题

Browser Insight 探针是否安全？会不会暴露用户的隐私？

  • Bi 探针非常安全，采用 JS 嵌码技术，和引入任何一个 JS 库的方式一样（例如：jQuery），Bi 探针采用 W3C 标准和接口抓取性能数据，不会抓取用户信息。
  • 在 APM 领域，OneAPM 是一家非常有经验并且发展非常稳定迅速的公司，不会让安全性、隐私性这种原则性问题影响公司未来的发展。

Browser Insight 探针会给服务器多大压力呢？

  • SaaS 级 Bi 探针不会给服务器造成任何压力，全部数据由 OneAPM 的 SaaS 平台接收。并且 JS 下载也只用几K大小，还没有一个 jQuery 大。

Browser Insight 注入页面的 JS 是什么加载方式？会影响页面响应吗？

  • Bi 的 JS 链接只是在初次访问被监控页面时才需要下载一次 JS 脚本，之后如果 10 天没有访问该页面的话，Bi 的 JS 脚本会自动清除,目前还没有出现因为 Bi 的嵌入 JS 的问题导致的页面加载报错的情况。如果实在担心 JS 的问题，推荐用户部署的时候使用纯文本的方式进行部署，就可以避免因为 Bi 的 JS 加载导致的页面问题.

Browser Insight 区域信息可以采集到哪一级别？

  • Bi可以采集到国内的省、地级市维度，国外的地理位置暂时只能到国家，之后会进一步具体到国外国家的下一级.

Browser Insight 可以采集到 Js 错误的哪些信息?

  • Bi 的 Js 错误可以采集到错误页面、错误次数、以及具体的错误信息包括：错误详情、堆栈信息、发生时间、错误类型、操作系统、浏览器以及版本等.

Browser Insight 探针的浏览器兼容问题?

  • OneAPM 的 Bi 探针可以兼容所有浏览器，包括 IE6 到 IE11、chrome、360 等各家浏览器
  • OneAPM 的 Bi 探针可以监控分析各个平台，包括 Pc 端、移动微信端、移动浏览器以及移动端 WebView，并且移动端安卓 4.3 版本以上还支持页面资源加载列表时序图。

Browser Insight CDN 资源监控能监控到哪些信息？

  • 地理、域名双维度监控分析网站资源加载，明确的看到各个地域的加速效果，不但让 CDN 加速在监控下一清二楚，更可以看到各个相关的域名，拒绝域名劫持和全站仿冒这两个问题。
  • 通过采集各个域名和地域下真实的用户访问页面的响应时间分布，对用户的真实访问状态了如指掌，弥补平均时间带来的不明确性.
  • 通过采集各个域名和地域下真实的用户访问页面的响应时间分布，对用户的真实访问状态了如指掌，弥补平均时间带来的不明确性。
  • 将 CDN 加速的资源分类统计，让用户一下子就能定位到只需要优化的地方，做到优化 CDN 资源有的放矢。
  • 设备，系统多角度全面统计分析，这样优化的时候还可以参考各个地域、各个域名的常用设备，不落下任何死角。
  • 资源性能耗时 TOP 10，全部基于不同地域、不同域名下的真实的用户访问，优化的是有更有目标.
